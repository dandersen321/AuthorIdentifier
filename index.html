Author Identifier

Apply Artificial Intelligence to Stylometry to identify authorship of documents.

Download this project as a .zip file Download this project as a tar.gz file

Executive Summary

It can be hard to determine the author of a document from the text alone. This problem may arise when checking for plagiarism or old documents whose authorship is in question. To solve such a problem, this tool uses a decision tree to learn the style of many authors. Then using this information it can analyze a document and determine with relatively high accuracy the original author of the work in question.

Description

For humans, it is difficult to analyze a document and recognize from the writing the author. This linguistic practice is known as Stylometry and has many useful applications. There are many ways this problem has been approached, but each typically consists of looking for similar patterns between an author's known works and the document being analyzed. This tool attempts to improve the accuracy of such a method by using a decision tree that can find complex patterns that might be undetected by traditional methods.

This implementation was used on a dataset of 24,428 books from the Gutenberg project (https://www.gutenberg.org/) but future releases may incorporate other media such as journals and emails. The tool will analyze each book to discover an author's style. If successful, it can then use this information to predict the author of future books it has not seen before. This tool has been written in python using the NLTK and sklearn packages for language analysis and AI learning respectively.

Why

There are several uses for the author identifier:

  1. Plagiarism: the tool could determine the probability that a paper under one author is actually written from another source
  2. Validity of works: the tool could determine the legitimacy of works whose authorship is in question.
  3. Discovering authors hidden under pen names: this tool could be used to help determine the author of books that have been written under a pen name
  4. Similar writing styles: the tool could help determine how close different author’s writing styles are

The Process

The Analysis Phase

From my initial research [1,2], I used 31 features in my analysis phase. They are:

  • apostrophesPerWord: how many apostrophes are used in each word
  • averageParagraphLength: the average number of sentences in each paragraph
  • averageWordLength: the average character length of each word
  • averageWordsPerSentence: the average words in each sentence
  • bigraphs: how often the two character sequence appears in a word
    • bigraph-co
    • bigraph-lc
    • bigraph-me
    • bigraph-we
  • digitFraction: how many characters are digits
  • numberOfWords: how many words the document contains
  • uppercaseFraction: how many words start with an uppercase character
  • whitespaceFraction: the ratio of the whitespace to words
  • WPT: (Words Per Thousand) how often a word occurs out of every thousand words
    • WPT-and
    • WPT-because
    • WPT-but
    • WPT-colon
    • WPT-comma
    • WPT-doubleHyphen
    • WPT-exclamationMark
    • WPT-however
    • WPT-hyphen
    • WPT-if
    • WPT-might
    • WPT-more
    • WPT-must
    • WPT-quote
    • WPT-semicolon
    • WPT-since
    • WPT-that
    • WPT-this
    • WPT-very

I calculated these features on each document then saved the results in a json file. Using this method, we don't have to analyze all the documents every time the program is run (a analysis on all 24,428 books takes about 8 hours).

Creating the decision tree

Results

Using this method I was able to predict the correct author with a relatively high degree of accuracy. This accuracy was correlated with two factors: the number of authors used and the minimum number of books each author must have written to be included in the dataset. The following graphs show the results.

We can see that as the number of authors increase, the accuracy decreases until it steadies at around 45% accuracy. So given a high school classroom size of 30 students we would expect a 75% accuracy in the prediction of anonymous documents and given a large college course of 250 students we would expect 45% accuracy.

A second but less impactful variable was the number of books an author needed to write to be included in the dataset. As we can see from the following graph, the more books we have to analyze from an author results in a small positive correlation to our accuracy.

Features of Importance

After looking at the results, I also explored which features improved the accuracy the most. The first way was to use only that feature and see how well I could predict the accuracy. This resulted in expectedly low accuracy, but shows which features are strong on their own.

I then looked at how the features combined to affect accuracy. I did this by starting with zero features, and then adding one random feature and computing it's accuracy in predicting authors. This resulted in the following data.

As we can see, the first features are the most significant but after several features have been added each additional ones have a smaller impact. We also see that certain features that did not do well on the single parameter test also added little or even hurt the accuracy when combined with other features.
